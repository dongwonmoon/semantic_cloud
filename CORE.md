# 의미 구름 기반 언어 모델 구상 문서
## 주제
**"attention은 네모나고 딱딱하다. 언어의 의미는 둥글고 흐리며, 문장 전체에 퍼진다."**  
이 직관을 바탕으로, **Transformer/Attention 중심의 언어 처리 관점**을 넘어서는 새로운 아키텍처 가능성을 탐색한다.

---

# 1. 문제의식

## 1.1 출발점
기존 Transformer의 attention은 매우 강력하지만, 다음과 같은 감각적/개념적 한계를 가진다고 느낀다.

- attention은 본질적으로 **토큰-토큰 관계 행렬**이다.
- 이 구조는 의미를 **"누가 누구를 얼마나 보는가"** 라는 방식으로 표현한다.
- 이 표현은 지나치게 **명확하고, 선명하고, 정적**이다.
- 그러나 실제 언어의 의미는 그렇게 작동하지 않는 것처럼 보인다.

내 직관에 따르면 언어는 다음과 같다.

- 단어의 의미는 **처음부터 날카롭게 결정되어 있지 않다**
- 의미는 **문맥 전체에 퍼져 있다**
- 단어들은 서로를 직접 "조회"하기보다 **전체 의미 상태를 변형**시킨다
- 해석은 순차적 누적이라기보다 **전체의 재배치**에 가깝다
- 긴 문장, 문단, 책은 **하나의 큰 의미 구름**으로 인식된다
- 핵심 의미는 마지막에 **응축되어 떠오른다**

즉, attention은 언어의 어떤 측면을 잘 잡지만,  
**언어의 퍼짐, 뭉침, 재해석, 중심 응축** 같은 현상을 다루기에는 구조적으로 딱딱해 보인다.

---

## 1.2 기존 관점의 문제점

### 문제점 A. 의미를 관계의 집합으로 본다
Transformer는 의미를 주로 **토큰 간 pairwise relation**의 집합으로 취급한다.  
하지만 내 직관에서는 의미는 관계 이전에 **전체 상태**다.

### 문제점 B. 연결이 너무 명확하다
attention은 softmax 이전부터 이미  
"이 토큰이 저 토큰을 얼마나 보느냐"라는 구조를 가진다.  
이 방식은 의미의 **경계가 흐린 상태**를 표현하기 어렵다.

### 문제점 C. 해석이 지나치게 조회 중심이다
언어 이해는 단순 조회(lookup)가 아니라,  
문장을 읽으며 내부 의미 상태가 **변형되고 휘고 재배치되는 과정**처럼 느껴진다.

### 문제점 D. next-token prediction 중심 해석이 직관과 어긋난다
사람이 문장을 쓸 때는 대개 **핵심 의미를 먼저 갖고 있고**,  
읽는 사람은 문장을 따라가며 그 의미를 복원한다.  
그런데 next-token prediction은 언어를 매 시점의 다음 단어 결정 문제로 환원한다.

### 문제점 E. 장거리 의존성을 "멀리 떨어진 연결"로 본다
내 직관에서 장거리 의존성은  
"멀리 있는 두 토큰이 직접 연결된다"가 아니라,  
**같은 큰 의미 구름에 속한다**에 가깝다.

---

# 2. 핵심 직관 정리

## 2.1 단어와 의미
- 단어는 점이 아니다
- 단어는 **구름** 또는 **덩어리**에 가깝다
- 중심은 비교적 뚜렷하지만 가장자리는 흐리다
- 단어 의미는 문맥에 따라 **늘어나고, 줄어들고, 휘어진다**
- 서로 다른 의미는 만나면 단순 연결이 아니라 **혼합되어 제3의 의미 구름**을 만들 수 있다

## 2.2 문장과 해석
- 문장은 토큰의 나열이 아니라 **하나의 큰 의미 구름**
- 문단은 문장보다 큰 구름
- 책은 문단보다 큰 구름
- 잘게 쪼개진 표면 아래에는 **하나의 중심 의미**가 있다
- 읽는 과정은 그 중심을 향해 전체 구름이 정렬되는 과정이다

## 2.3 해석의 동역학
- 해석은 매 순간 조금씩 바뀔 수도 있고,
- 어떤 단어를 만났을 때는 전체가 **다시 태어나는 수준으로 재배열**될 수도 있다
- 의미는 고정된 참조가 아니라 **팽창, 수축, 휨, 소용돌이, 찢김, 재결합** 같은 변형을 겪는다

---

# 3. 핵심 가설

## 가설 1
언어 이해의 본질은 **토큰 간 참조**보다 **전역 의미 상태의 변형**에 가깝다.

## 가설 2
의미를 pairwise attention matrix로 표현하는 대신,  
**전역 의미 분포(global semantic field / cloud)** 로 표현하면  
언어의 모호성, 함축, 분위기, 장거리 일관성을 더 자연스럽게 다룰 수 있다.

## 가설 3
좋은 언어 모델은 "다음 단어를 맞히는 기계"이기 전에,  
입력으로부터 **중심 의미를 응축하고 안정화하는 시스템**이어야 한다.

## 가설 4
장거리 의존성은 explicit한 먼 연결이 아니라,  
멀리 떨어진 부분들이 **동일한 의미장에 속함**으로써 자연스럽게 설명될 수 있다.

## 가설 5
아포리즘, 비유, 함축적 문장처럼  
마지막에 전체가 재해석되는 유형의 언어에서는  
attention 기반 구조보다 **상태 재배치 중심 구조**가 더 자연스러운 내부 표현을 만들 수 있다.

---

# 4. 이 아이디어가 중요한 이유

## 4.1 단순 성능 개선 이상의 의미
이 아이디어는 단순히 "attention을 조금 고치자"가 아니다.  
그보다 더 깊게, **언어를 무엇으로 보는가**에 대한 관점을 바꾸려는 시도다.

## 4.2 내부 표현의 자연스러움
내 목표는 무조건 perplexity를 이기는 것이 아니다.  
오히려 다음을 중요하게 본다.

- 내부 구조가 더 직관적인가?
- 의미가 더 자연스럽게 응축되는가?
- 긴 문맥에서 전체가 하나의 상태처럼 유지되는가?
- 재해석 현상을 더 납득 가능하게 설명하는가?

## 4.3 향후 가능성
만약 이 방향이 유효하다면 다음 영역에 시사점이 있을 수 있다.

- 아포리즘, 시, 문학, 철학 텍스트 해석
- 긴 문맥 요약
- 저자의 핵심 주장 복원
- 의미 중심적 생성
- 보다 인간적인 독해 과정 모델링

---

# 5. 새 관점의 핵심 정의

## 5.1 의미 구름 (Semantic Cloud)
문장 전체에 퍼져 있는 의미 상태.  
선명한 연결들의 집합이 아니라,  
**중심/후광/밀도/혼합/불확실성**을 가진 분포적 상태.

## 5.2 중심 의미 (Core Meaning)
구름 내부에서 응축되어 떠오르는 핵심 의미.  
쓰는 사람은 이를 먼저 갖고 있을 가능성이 높고,  
읽는 사람은 문장을 다 읽으며 이를 복원한다.

## 5.3 변형 (Deformation)
새로운 입력이 들어왔을 때 내부 상태가 바뀌는 방식.  
예시:
- 팽창
- 수축
- 휨
- 소용돌이
- 찢김
- 재결합
- 중심 이동
- 후광 확대/축소

## 5.4 재배치 (Reconfiguration)
입력이 누적되며 상태가 점진적으로 수정되거나,  
어느 순간 전체가 한 번에 다시 정렬되는 현상.

## 5.5 응축 (Condensation)
퍼진 의미 상태 속에서 핵심 중심이 떠오르는 과정.  
내 모델의 출력은 단순 next-token보다 이 응축 현상에 더 가까워야 한다.

---

# 6. 목표

## 6.1 최상위 목표
attention/transformer를 그대로 수정하는 것이 아니라,  
**의미장의 변형과 응축**을 중심으로 하는 대체 구조를 구상한다.

## 6.2 하위 목표
1. 직관을 일관된 개념 체계로 정리한다
2. 구현 가능한 내부 표현 방식을 고른다
3. 최소 프로토타입을 만든다
4. 어떤 유형의 언어에서 강점을 보이는지 확인한다
5. 실패하더라도 어떤 부분이 핵심 병목이었는지 해석 가능하도록 한다

# 7. 해결 방안 후보

# 해결 방안 1. 가우시안 혼합 의미 구름 모델
가장 현실적인 1차 후보.

## 개념
전역 의미 상태를 여러 개의 구름으로 근사한다.  
각 구름은 예를 들어 다음 값을 가진다.

- 중심
- 퍼짐 정도
- 강도
- 혼합 비율

즉, 전체 의미는 **Gaussian-like blobs의 mixture** 로 표현된다.

## 장점
- "중심은 뚜렷하고 가장자리는 흐리다"는 직관과 잘 맞는다
- 의미가 섞여 제3의 구름이 생기는 현상을 비교적 자연스럽게 구현할 수 있다
- 수학적으로 상대적으로 다루기 쉽다
- 최소 프로토타입 후보로 적합하다

## 단점
- 실제 의미가 가우시안으로 잘 근사되지 않을 수 있다
- 복잡한 문맥에서는 구름 수가 많이 필요할 수 있다
- 매우 정교한 의미 변형을 담기엔 단순할 수 있다

## 구현 이미지
- 입력 token은 내부 상태를 직접 설명하지 않고 **변형 신호**를 생성한다
- 각 step에서 구름들의 중심/분산/가중치를 업데이트한다
- 특정 주기마다 구름을 병합하거나 분리한다
- 최종적으로 가장 강한 중심을 읽어 핵심 의미로 사용한다

---

# 해결 방안 2. 입자 기반 의미장 모델
좀 더 동적인 버전.

## 개념
전역 상태를 연속장으로 가정하되, 실제 계산은 많은 의미 입자들의 집합으로 근사한다.  
입력 token은 입자들의 분포와 운동을 바꾼다.

## 장점
- "쫀득쫀득 변형됨", "소용돌이", "찢어졌다 붙음" 같은 직관과 매우 잘 맞는다
- 복잡한 의미 변화 과정을 시각화하기 좋다
- 지역적 뭉침과 전역적 확산을 동시에 표현할 가능성이 있다

## 단점
- 학습과 구현이 어렵다
- 계산량이 클 수 있다
- 불안정할 가능성이 높다

## 구현 이미지
- 의미 입자들의 좌표/밀도/상태를 유지
- token이 들어오면 특정 방향으로 힘(force)을 가함
- 일정 step마다 입자들이 응집/확산/재정렬
- 최종적으로 입자 분포의 attractor를 읽어 의미 중심 추출

---

# 해결 방안 3. 암시적 의미장 함수 모델
가장 이상적이지만 초기 구현은 어려움.

## 개념
의미를 고정된 벡터나 구름 집합이 아니라  
하나의 함수 형태로 둔다.  
즉, "현재 문장의 의미 밀도 함수" 자체를 내부 상태로 본다.

## 장점
- "상태의 분포"라는 직관을 가장 잘 반영한다
- 문장/문단/책까지 확장되는 계층적 구조와도 잘 맞는다
- 토큰 단위 내부 표현에서 많이 벗어날 수 있다

## 단점
- 매우 추상적이다
- 구현 난도가 높다
- 초기 실패 시 왜 실패했는지 파악이 어려울 수 있다

## 구현 이미지
- 상태를 직접 벡터로 들고 있지 않고, 작은 신경망/잠재 함수가 의미장을 나타냄
- 새 입력은 이 함수를 parameter update처럼 변형함
- 읽기(readout)는 함수의 peak, basin, concentration 등을 통해 수행

---

# 해결 방안 4. 계층적 의미 응축 모델
문장-문단-책 구조를 노린다.

## 개념
모든 입력은 하위 수준에서 상위 수준 의미장으로 점차 응축된다.

- token → 구
- 구 → 문장
- 문장 → 문단
- 문단 → 장
- 장 → 책 전체 중심 의미

## 장점
- "책 전체가 하나의 큰 구름"이라는 직관과 강하게 맞는다
- 긴 문맥 처리에서 유리할 가능성이 있다
- 단순 sequence보다 구조적 읽기 과정에 가깝다

## 단점
- 계층 경계를 어떻게 잡을지 모호하다
- 초기 실험이 복잡해질 수 있다
- 우선은 하위 수준 프로토타입 후 확장하는 편이 낫다

---

# 8. 내가 현재 가장 유력하게 보는 1차 구현안

## 추천안
**해결 방안 1: 가우시안 혼합 의미 구름 모델**

이유는 다음과 같다.

1. 네 직관을 가장 덜 훼손한다
2. "중심 + 흐린 가장자리"를 자연스럽게 표현한다
3. "혼합되어 제3의 의미가 생김"을 모델링하기 쉽다
4. 너무 추상적이지 않아 최소 실험이 가능하다
5. 실패해도 왜 실패했는지 비교적 해석 가능하다

# 9. 제안 아키텍처 스케치

## 이름(가칭)
**Cloud Field Reconfiguration Model (CFRM)**

## 핵심 구성
### 9.1 입력 계층
- 입력은 token sequence
- token은 내부 표현의 최종 단위가 아니라 **표면 자극**
- 각 token은 의미장에 줄 변형 벡터/신호를 생성

### 9.2 전역 의미 상태
- 문장 전체를 나타내는 하나의 전역 상태 `S`
- `S`는 여러 의미 구름들의 혼합 상태로 표현

### 9.3 변형 연산자
새 token `x_t`가 들어오면

`S_{t+1} = Transform(S_t, x_t)`

이때 Transform은 lookup이 아니라 deformation에 가깝다.

가능한 연산:
- 중심 이동
- 분산 확대/축소
- 구름 병합
- 구름 분리
- 밀도 재가중치
- 전체 상태의 국소적 휨

### 9.4 재배치 단계
일정 단계마다 또는 특정 신호가 강할 때,
의미 구름 전체를 다시 정렬한다.

이 단계는 네가 말한
"어느 순간 전체가 다시 태어나는 수준의 재해석"
을 모방한다.

### 9.5 응축 및 출력
최종적으로
- 가장 강한 중심
- 주변 불확실성
- 다중 후보 중심 여부
를 읽는다.

출력은 next-token보다 먼저
**핵심 의미 요약 / 중심 상태 추출** 쪽이 더 어울린다.

# 10. 학습 목표 설계

## 10.1 지양할 것
- next-token prediction만을 유일 목표로 삼는 것

이건 내 아이디어의 존재 이유를 약화시킬 수 있다.

## 10.2 권장 목표
### 목표 A. 중심 의미 복원
문장을 보고 핵심 의미를 복원하게 한다.

예:
- 한 줄 요약
- 핵심 주장 추출
- 문장의 중심 의미 라벨 예측

### 목표 B. 재해석 일관성
문장 중간 상태와 최종 상태 사이의 재구성 차이를 학습한다.

예:
- 앞부분만 읽었을 때의 중심 후보
- 끝까지 읽었을 때 중심 후보 변화량

### 목표 C. 표현은 다르지만 중심이 같은 문장 정렬
의역/패러프레이즈 문장들이 비슷한 중심 의미를 갖도록 학습.

### 목표 D. 계층 응축
문장 → 문단 → 문서 수준에서 의미 중심이 일관되게 이어지도록 학습.

# 11. 실험 우선순위

## 1단계. 개념 검증
목표:
- 이 구조가 최소한 "말이 되는 내부 동역학"을 만드는지 확인

확인할 것:
- 의미 구름이 정말 형성되는가?
- 중심/후광 구조가 나타나는가?
- 특정 단어에서 전체 재배치가 발생하는가?

## 2단계. 직관적 사례 실험
대상:
- 아포리즘
- 반전 문장
- 마지막 단어에서 의미가 뒤집히는 문장
- 은유적 표현

관찰:
- attention류보다 상태 변화 해석이 자연스러운가?

## 3단계. 소규모 정량 실험
지표 예시:
- 중심 의미 복원 정확도
- 문장 전체 요약 일관성
- 중간 상태와 최종 상태의 해석 차이 추적 가능성
- 긴 문맥 핵심 유지력

## 4단계. 비교 실험
비교 대상:
- 소형 Transformer baseline
- 동일한 토크나이저, 비슷한 파라미터 수
- 동일한 데이터셋

비교 포인트:
- perplexity만이 아니라
- 내부 표현의 해석 가능성
- 재해석 상황에서의 안정성
- 긴 문맥에서의 중심 보존

# 12. 기대 효과

## 기대 1
문장을 단순 연결망이 아니라 **하나의 상태**로 바라보게 된다.

## 기대 2
모호성, 함축, 분위기 같은 현상을 더 자연스럽게 다룰 수 있다.

## 기대 3
장거리 의존성을 direct link 대신 **공유된 의미장**으로 설명할 수 있다.

## 기대 4
문장/문단/문서 전체를 아우르는 계층적 의미 응축 모델로 발전할 가능성이 있다.

# 13. 예상되는 실패 원인

## 실패 원인 A. 지나치게 추상적임
좋은 비유이지만 계산 구조로 내릴 때 손실이 클 수 있다.

## 실패 원인 B. 너무 부드러워져서 정밀성이 무너짐
구름과 혼합만 강조하면,  
정확한 기호 처리나 명시적 참조가 필요한 문제에서 약할 수 있다.

## 실패 원인 C. 내부 표현은 예쁜데 성능이 안 나옴
해석 가능성은 좋아도 실용적 성능이 떨어질 수 있다.

## 실패 원인 D. 결국 Transformer가 이미 일부를 암묵적으로 하고 있을 수 있음
multi-head, residual, deep layers가  
이미 sharp/diffuse, local/global 역할 분담을 어느 정도 수행할 수도 있다.

## 실패 원인 E. 학습 목표가 잘못 잡힐 수 있음
next-token prediction만으로는 이 구조의 장점을 제대로 끌어내기 어려울 수 있다.

# 15. 지금 시점의 중간 결론

나는 현재 다음처럼 정리할 수 있다.

> 언어 이해는 token 간 explicit한 참조보다,  
> 입력이 하나의 전역 의미 분포를 계속 변형시키고,  
> 그 과정 끝에 중심 의미가 응축되는 현상에 더 가깝다.

이 관점을 밀어붙이면,
attention 기반 구조의 부드러운 변형이 아니라  
**전역 의미장 + 변형 연산 + 재배치 + 응축**을 중심으로 한  
새로운 언어 모델 아키텍처 구상이 가능하다.
